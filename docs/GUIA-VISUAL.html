4 Keys — o que ler, o que nunca pedir

Key Agent faz o quê? Para quê
OPEN_ROUTER_API_KEY Lê da env do Windows — não pede no chat Chat do app (produto) — padrão smoke
OPENROUTER_API_KEY Lê se existir — não pede Fallback .env.local do app
ANTHROPIC / OPENAI / GEMINI_API_KEY pay-as-you-go Não (como motor do coding agent) Use a subscription do CLI/app
Você seta OPEN_ROUTER_API_KEY uma vez no Windows. O app resolve sozinho. Smoke: GET /api/health. Se faltar: HANDOFF “setar no Windows” — sem colar o secret no chat.

5 Bernstein — o que é de verdade

Bernstein não é um LLM. É um maestro em Python: recebe um goal, parte em tarefas, dispara Claude Code / Codex / Gemini CLI (suas subs) cada um em git worktree isolado, mostra progresso, roda build/test e só mergeia o que passou. Explicação longa: docs/BERNSTEIN-EXPLICADO.md
Pipeline interno
1 Decomposegoal → tasks
→
2 Spawn1 CLI / worktree
→
3 Verifynpm run build
→
4 Mergesó o que passou

É o maestro

  • Orquestra vários coding CLIs
  • Progresso: bernstein live / dashboard
  • Audit log do que cada agent fez
  • Bom em L1 code-heavy (B1/B4)

Não é o kanban

  • Não substitui QUEUE / HANDOFF
  • Não é Grok / L0 content
  • Não é OpenRouter do WaifuChat
  • Não é Langflow / CLI-Anything
Camada Ferramenta Papel
Workbench QUEUE · HANDOFF · CLAIMS Kanban + handoff entre você e várias subs (incl. Grok)
Bernstein bernstein + CLIs Maestro que dispara e monitora coding agents em paralelo
OpenRouter OPEN_ROUTER_API_KEY Cérebro do app pro usuário final

7 Erros comuns

Errado Certo
Começar app novo sem scorecard L0/P0 → go → só então L1
Uma sessão = 5 features Uma sessão = 1 job da QUEUE
Pedir OPENROUTER key no chat Ler OPEN_ROUTER_API_KEY do sistema
API Anthropic/OpenAI pro agent codar Subscription Claude Code / Codex / etc.
Rate limit → recomeçar do zero L2: HANDOFF → outra sub → mesmo job
Declarar pronto sem build npm run build + atualizar HANDOFF
Bernstein para tudo Bernstein só L1 code; L0/Grok no workbench
Instalar Langflow/CLI-Anything/MALLM “pra ver progresso” Progresso = workbench + opcional bernstein live

8 Checklist antes de cada sessão

Em uma frase: Workbench orquestra humanos e subs; prompts fecham 1 job; OpenRouter roda o app via env; Bernstein é opcional para ver progresso de L1 code — nunca misture carteira de coding com carteira de produto.
